# 管理員操作手冊

[English](en/admin-guide.md) · [文件首頁](README.md) · [使用者操作](user-guide.md)

本手冊聚焦提案審核、處理狀態、公告與日常管理。權限由後端規則決定；畫面上出現管理操作不代表可以繞過內容的可見範圍。

## 開始前檢查

- 使用屬於允許網域、且列入管理員名單的帳號登入。
- 權限剛調整時重新登入，讓新的角色資訊生效。
- 先確認各分類的公開範圍、作者顯示、附議門檻與回覆期限。
- 不在公開內容或截圖中暴露私密提案與使用者資料。

## 審核提案

1. 從待審核內容開啟提案。
2. 檢查分類、公開適切性、個人資訊與媒體內容。
3. 選擇審核通過，或填寫具體且可採取行動的未通過原因。
4. 核准後確認公開範圍、作者顯示與附議設定符合預期。

審核只決定內容是否進入後續流程，不應用來替代校規、申訴或緊急事件的正式處理管道。

## 更新處理狀態

將已受理提案標記為處理中；結案時選擇已完成或不可行，並填寫使用者看得懂的結果。不要用狀態取代與作者必要的私密溝通，也不要填入尚未確認的承諾。

| 狀態 | 管理重點 |
| --- | --- |
| 待審核 | 優先檢查公開風險與分類是否正確 |
| 處理中 | 保持進度與回覆期限可追蹤 |
| 已完成 | 說明完成內容與可驗證結果 |
| 不可行 | 說明限制、原因與可能的替代方向 |

## 發布與管理公告

公告發布後不可編輯，且可能透過站內通知與 Web Push 廣播。發布前應再次確認標題、Markdown、圖片、連結、日期與對象；若內容有誤，依平台現有能力刪除後重新發布。

## Dashboard 與日常管理

Dashboard 用於查看平台統計、分類成果、近期活動與背景工作異常，但不是稽核系統。建議固定檢查：

- 待審核內容與即將到期的回覆。
- 分類成果、近期活動與異常變化。
- 通知、圖片與外部同步是否持續正常。

異常時依[維運手冊](operations.md)查看紀錄與供應商狀態；操作或顯示問題先看[故障排除](troubleshooting.md)。

## 安全與交接

- 管理員異動時同步調整名單並要求重新登入。
- 不共用管理員帳號，不把正式環境憑證貼入提案或留言。
- 發現越權、資料外洩或可疑登入時，停止操作並依[安全回報流程](security.md#回報安全問題)處理。
- 交接時記錄未完成審核、處理中提案與已知異常，不只交接帳號。
